# Item 10: From Next.js to Astro: A Page Size Comparison

Source: [https://alan.norbauer.com/articles/astro-vs-nextjs-page-size/](https://alan.norbauer.com/articles/astro-vs-nextjs-page-size/)

Summary:
A developer compares the page size of a static site after migrating from Next.js (with RSC) to Astro. The Astro version results in significantly smaller HTML and JS payloads, with all other factors kept equal. The post emphasizes that Astro is better suited for small, static sites from a size perspective, though Next.js isn’t inherently bad.

Key takeaways:
- Astro produces much smaller page sizes than Next.js for static sites.
- Next.js with RSC may be overkill for simple static content.
- Choice of framework should match project requirements and optimization goals.
