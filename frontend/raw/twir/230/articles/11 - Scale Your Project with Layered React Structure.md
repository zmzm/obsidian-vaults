# Item 11: Scale Your Project with Layered React Structure

Source: [https://playfulprogramming.com/posts/layered-react-structure](https://playfulprogramming.com/posts/layered-react-structure)

Summary:
This article proposes the "Layered React Structure" (LRS) for organizing large React projects, inspired by Angular's opinionated style guides. LRS emphasizes separating "smart" (business logic) and "dumb" (presentation) components, with clear guidelines for state and logic placement. The approach aims to improve maintainability, testability, and developer onboarding by establishing consistent project structure.

Key takeaways:
- LRS advocates for clear separation between business logic and UI components.
- "Dumb" components may have UI state but not business logic; "smart" components handle data fetching, effects, and error handling.
- Consistent structure reduces bugs, eases testing, and accelerates development.
- The methodology is refined from real-world, large-scale React project experience.
