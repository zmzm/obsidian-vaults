# Item 7: Keeping Costs in Check When Hosting Next.js on Vercel

Source: [https://www.thisdot.co/blog/keeping-costs-in-check-when-hosting-next-js-on-vercel](https://www.thisdot.co/blog/keeping-costs-in-check-when-hosting-next-js-on-vercel)

Summary:
This post explains Vercel’s pricing model for Next.js apps, breaking down fixed (per-seat) and usage-based (serverless function invocations, execution time) costs. It offers practical strategies for minimizing usage charges, such as batching server actions, optimizing API routes, leveraging caching, and configuring middleware matchers to avoid unnecessary invocations.

Key takeaways:
- Vercel charges per team member (DX Platform) and per serverless usage (function invocations, execution time).
- Every backend request, including server actions and React server component renders, counts as an invocation.
- Strategies to reduce costs: batch actions, minimize backend calls, use caching, and optimize middleware.
- Static content and correct API route usage can further limit serverless invocations.
