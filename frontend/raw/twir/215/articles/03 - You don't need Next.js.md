# Item 3: You don't need Next.js

Source: [https://www.comfydeploy.com/blog/you-dont-need-nextjs](https://www.comfydeploy.com/blog/you-dont-need-nextjs)

Summary:
The ComfyDeploy team migrated their dashboard from Next.js to plain React with TanStack Router and Rspack, resulting in dramatically faster build times, hot reloads, and improved developer happiness. The article details the pain points with Next.js (slow builds, complex API/server actions, high costs) and the benefits and trade-offs of moving to a simpler React setup. While some Next.js features were lost (co-location, caching, pre-rendering, server actions), the migration led to clearer architecture and more intentional API design.

Key takeaways:
- Migration reduced build times from minutes to seconds and improved local dev speed.
- Next.js's complexity and server features were unnecessary for their dashboard use case.
- Trade-offs included losing built-in caching, co-location, and server actions, but these led to better architecture.
- Next.js remains useful for landing pages/SEO, but not always for product dashboards.
