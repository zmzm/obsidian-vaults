# Item 8: Why We Moved off Next.js

Source: [https://documenso.com/blog/why-we-moved-off-next-js](https://documenso.com/blog/why-we-moved-off-next-js)

Summary:
Documenso details their migration from Next.js to React Router (with Vite), citing issues with debugging Server Actions, slow hot module reloads, and poor developer experience as the app scaled. The switch resulted in faster builds, improved productivity, and greater flexibility, though some pain points were encountered due to the relative newness of React Router’s latest version.

Key takeaways:
- Next.js App Router and Server Actions caused debugging and build issues at scale.
- Turbopack did not resolve development speed problems.
- Migrating to React Router and Vite led to significant improvements in developer experience and build times.
- The team prefers explicit, less “magical” routing and values Vite’s plugin ecosystem.
