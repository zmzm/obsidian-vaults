# Item 3: React Frameworks and Server-Side Features: Beyond Client-Side Rendering

Source: [https://certificates.dev/blog/react-frameworks-and-server-side-features-beyond-client-side-rendering?friend=TWIR](https://certificates.dev/blog/react-frameworks-and-server-side-features-beyond-client-side-rendering?friend=TWIR)

Summary:
This article reviews the evolution of React frameworks, focusing on server-side features like SSR (with streaming), static pre-rendering, hydration, Server Components, and Server Functions. It explains how frameworks such as Next.js, React Router, and TanStack Start simplify full-stack React development, and provides code examples for key APIs and patterns. The guide emphasizes the importance of choosing frameworks for production apps and clarifies how modern SSR and pre-rendering work under the hood.

Key takeaways:
- Modern React frameworks automate bundling, routing, SSR, code splitting, and deployment, reducing manual setup.
- SSR with streaming and static pre-rendering improve performance and SEO; frameworks handle most configuration.
- Hydration connects server-rendered HTML to client interactivity, with APIs like hydrateRoot and suppressHydrationWarning.
- Server Components and Functions enable zero-bundle-impact server logic and direct access to backend resources.
