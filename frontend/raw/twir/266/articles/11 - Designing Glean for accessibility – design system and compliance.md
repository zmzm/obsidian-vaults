# Item 11: Designing Glean for accessibility – design system and compliance

Source: [https://www.glean.com/blog/designing-glean-for-accessibility-design-compliance](https://www.glean.com/blog/designing-glean-for-accessibility-design-compliance)

Summary:
Glean describes its journey to making accessibility a default by standardizing components and design tokens, powered by Base UI. The migration improved screen reader support, keyboard navigation, and focus management, while enforcing consistent APIs and limiting ad-hoc styling. The team also completed a VPAT for formal accessibility compliance, making accessibility a core product principle.

Key takeaways:
- Migration to Base UI and standardized tokens/components ensures accessibility by default.
- Components now have focused APIs and limited styling overrides, reducing regressions.
- Improved screen reader, keyboard, and focus support is built-in.
- Formal accessibility compliance (VPAT) achieved for customer transparency.

Recommendation: Read fully (read fully if building accessible design systems or migrating to Base UI)
