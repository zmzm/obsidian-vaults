# Item 5: JSX Over The Wire

Source: [https://overreacted.io/jsx-over-the-wire/](https://overreacted.io/jsx-over-the-wire/)

Summary:
This article explores the idea of sending JSX components directly from the server as API responses, rather than just JSON data. It discusses how this approach inverts the traditional client-driven data fetching model, letting the server "call" components by returning them as responses. The piece examines the implications for data modeling, component design, and the separation between models and view models.

Key takeaways:
- Proposes treating API endpoints as "parent components" that return JSX, not just data.
- Highlights the distinction between model data and view models tailored for UI.
- Suggests this approach can simplify data flow and component composition in some scenarios.
- Raises questions about the boundaries between API and UI logic.
