# Item 5: How to Use Action Routes in React Router

Source: [https://sergiodxa.com/tutorials/use-action-routes-in-react-router](https://sergiodxa.com/tutorials/use-action-routes-in-react-router)

Summary:
This tutorial introduces the "Action Routes" pattern for React Router, centralizing logic for resource actions (create, update, delete) in dedicated files. It demonstrates organizing actions for reuse, handling server/client logic, validation, i18n, and post-action effects like toasts or redirects. The approach improves maintainability and consistency, especially in apps with repeated or shared actions.

Key takeaways:
- Organize resource actions in a dedicated folder for reuse across routes/components.
- Supports both server and client logic in the same action file.
- Encourages clear naming conventions (e.g., noun-verb) and integration with validation/auth/i18n.
- Enables handling of client-side effects after server actions.
