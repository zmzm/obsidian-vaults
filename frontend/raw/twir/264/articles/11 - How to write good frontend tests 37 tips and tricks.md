# Item 11: How to write good frontend tests: 37 tips and tricks

Source: [https://howtotestfrontend.com/resources/how-to-write-good-frontend-tests](https://howtotestfrontend.com/resources/how-to-write-good-frontend-tests)

Summary:
The author shares practical, experience-based advice for writing maintainable and effective frontend tests, focusing on clarity, avoiding implementation details, and using appropriate query methods. The tips emphasize testing user-facing behavior, splitting tests by concern, and leveraging semantic queries for better readability and accessibility.

Key takeaways:
- Write focused tests that clearly state their intent and avoid mixing unrelated assertions.
- Avoid testing implementation details; focus on observable user behavior.
- Use semantic queries (getByRole, getByLabelText, etc.) for more robust and accessible tests.
- Don’t test third-party or framework internals; test your own logic and UI.
