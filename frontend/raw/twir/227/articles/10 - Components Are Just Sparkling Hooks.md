# Item 10: Components Are Just Sparkling Hooks

Source: [https://www.bbss.dev/posts/sparkling-hooks/](https://www.bbss.dev/posts/sparkling-hooks/)

Summary:
This essay argues that React components are essentially specialized hooks, as both follow the rules of hooks and can be composed similarly. The author demonstrates how components and hooks can be interchanged, and how headless hooks can expose state and handlers for more flexible composition, rather than just returning React nodes.

Key takeaways:
- Components and hooks share core mechanics; components are a subtype of hooks.
- Headless hooks allow for greater flexibility by exposing state and logic, not just UI.
- Encourages thinking in terms of composable logic, not just UI trees.
- Useful for advanced component abstractions and custom hooks.
