# TanStack Start Migration Drivers

This source captures why some teams are leaving Next.js for TanStack Start, with emphasis on simplicity, explicitness, and reduced framework overhead.

## Summary

- The article frames TanStack Start as closer to plain React and easier to reason about.
- Next.js is presented as increasingly complex, rapidly changing, and too tightly coupled to Vercel-oriented assumptions.
- The appeal of TanStack Start is framed around type-safe routing, explicit boundaries, and lower cognitive overhead.

## Why This Source Matters

- It gives the framework comparison branch a stronger articulation of the migration case away from Next.js.
- It helps explain that this is not only about performance, but also about mental model and operating cost.

## Caveats

- The source is opinionated and likely reflects a self-selecting subset of teams.
