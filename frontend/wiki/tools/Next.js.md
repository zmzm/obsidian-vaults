# Next.js

Next.js is one of the main framework hubs in this vault. This page should collect platform changes, API additions, runtime capabilities, and architectural implications for React applications.

## Key Ideas

- The important signal is not only individual releases but the overall direction of the framework.
- Adapter APIs, offline APIs, caching, i18n, and infrastructure choices are especially relevant because they shape application architecture.
- Next.js often acts as an indicator of where the production React ecosystem is moving.

## Practical Significance

- This page should become the main overview for Next.js changes inside the vault.
- It is a good place to connect release notes, RFCs, PRs, and durable patterns.

## Current Signals

- The current raw layer shows Next.js as both a platform-direction story and a practical architecture story.
- Stable adapters, `use cache`, App Router ergonomics, and debugging/tooling improvements are all already represented in the source set.
