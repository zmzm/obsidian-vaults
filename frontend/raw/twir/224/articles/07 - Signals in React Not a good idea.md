# Item 7: Signals in React? Not a good idea

Source: [https://www.felgus.dev/blog/signals-in-react](https://www.felgus.dev/blog/signals-in-react)

Summary:
The article critiques attempts to bring fine-grained reactivity (signals) into React, arguing that React’s coarse-grained, component-based model is fundamentally incompatible with signals’ direct UI updates. Practical and conceptual issues arise, including awkward data access patterns and unpredictable behavior due to React’s rendering model. The author suggests using frameworks built for signals (like Solid.js) if fine-grained reactivity is needed.

Key takeaways:
- React’s architecture is not designed for fine-grained reactivity via signals.
- Community signal libraries for React introduce complexity and potential bugs.
- React’s predictable, component-based updates are a core strength.
- Prefer frameworks like Solid.js for signal-based reactivity.
