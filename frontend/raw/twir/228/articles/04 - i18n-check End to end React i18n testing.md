Summary:
i18n-check is a tool for validating and testing internationalization (i18n) in React apps, supporting both locale file checks and codebase analysis for react-intl and react-i18next. It helps identify missing, invalid, or out-of-sync translation keys, and can be integrated into project scripts for automated checks. The article provides setup instructions and usage scenarios for maintaining translation integrity.

Key takeaways:
- Supports both ICU and i18next formats; analyzes codebases using react-intl/react-i18next.
- Detects missing, invalid, or mismatched translation keys between source and target locales.
- Can be run via CLI or package scripts; flexible for different project structures.
- Helps prevent runtime translation errors and keeps i18n in sync.
