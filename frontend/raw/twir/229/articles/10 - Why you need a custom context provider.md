# Item 11: Why you need a custom context provider

Source: [https://gabrielpichot.fr/blog/why-you-need-a-custom-context-provider/](https://gabrielpichot.fr/blog/why-you-need-a-custom-context-provider/)

Summary:
The article argues that using “naked” context providers in React can lead to unnecessary re-renders and performance issues. By encapsulating state management in custom context provider components, teams can better control re-renders and achieve cleaner separation of concerns, especially in large or deeply nested component trees.

Key takeaways:
- Direct use of Context.Provider can cause all consumers to re-render on value changes, even unrelated components.
- Custom context providers encapsulate state logic and limit unnecessary re-renders.
- This pattern leads to improved performance and maintainable codebases.
- The article provides practical code comparisons and recommendations.
