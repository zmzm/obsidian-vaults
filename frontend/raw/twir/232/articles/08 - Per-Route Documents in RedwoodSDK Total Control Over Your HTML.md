# Item 11: Per-Route Documents in RedwoodSDK: Total Control Over Your HTML

Source: [https://rwsdk.com/blog/redwoodsdk-multiple-documents](https://rwsdk.com/blog/redwoodsdk-multiple-documents)

Summary:
RedwoodSDK allows each route to define its own HTML document structure, unlike most frameworks that use a global template. Developers can choose whether to hydrate with React, use static HTML, or implement custom behaviors per route. This flexibility enables optimized payloads and tailored experiences for different parts of an app, especially on Cloudflare’s edge.

Key takeaways:
- Each route can have a distinct document structure and hydration strategy.
- Enables static, dynamic, or realtime pages within the same app.
- Gives developers full control over HTML and React integration per route.
