# Item 6: Armin's TanStack Router Notes

Source: [https://gist.github.com/mitsuhiko/555b45f020a8d7e389516b51f6d03534](https://gist.github.com/mitsuhiko/555b45f020a8d7e389516b51f6d03534)

Summary:
This gist provides critical feedback on TanStack Router from the perspective of an experienced developer. It discusses TypeScript complexity, Suspense handling, context management, lack of named routes, code generation quirks, and the clunkiness of code-based routes, especially for large or dynamic apps.

Key takeaways:
- TypeScript integration is powerful but can be overwhelming and inflexible.
- Suspense is handled automatically, which may be confusing for custom loading logic.
- Context management differs from React Query, leading to awkward patterns.
- Lack of explicit route naming and dynamic routing support can hinder large-scale apps.
