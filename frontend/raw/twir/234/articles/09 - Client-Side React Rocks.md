# Item 9: Client-Side React Rocks

Source: [https://marmelab.com/blog/2025/05/09/client-side-react-rocks.html](https://marmelab.com/blog/2025/05/09/client-side-react-rocks.html)

Summary:
This article argues that client-side React provides a superior developer experience for many use cases compared to React Server Components (RSC), especially for highly interactive apps. Using the example of a data grid, it highlights composability, reusability, and performance benefits of client-side APIs, such as using components for column definitions and leveraging React’s context and effects. The limitations of RSC for certain UI patterns are also discussed.

Key takeaways:
- Client-side React excels at composability and reusability for interactive UIs.
- Component-based APIs are more ergonomic than JSON-based configs.
- RSC is not always suitable for complex, dynamic client-side patterns.
- Performance optimizations (e.g., memoization) are easier with client-side React.
