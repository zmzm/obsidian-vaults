# Item 4: Bringing React's <ViewTransition> to vanilla JS

Source: [https://plainvanillaweb.com/blog/articles/2025-06-12-view-transitions/](https://plainvanillaweb.com/blog/articles/2025-06-12-view-transitions/)

Summary:
This article examines how React’s <ViewTransition> feature handles view transitions and explores how similar effects can be achieved in vanilla JavaScript. It highlights the complexities React abstracts away—such as batching, queuing, and feature detection—and demonstrates a simplified, framework-agnostic approach. The piece provides practical insights into the browser’s native view transition API and discusses the trade-offs between React’s “magic” and explicit vanilla implementations.

Key takeaways:
- React’s <ViewTransition> component manages unique naming, batching, and safe queuing of transitions, simplifying complex UI flows.
- The browser’s native view transition API is powerful but requires careful handling of async updates, feature detection, and user input blocking.
- A vanilla JS approach can replicate much of React’s functionality with explicit queueing and batching logic.
- Understanding both approaches helps developers choose the right level of abstraction and control for their needs.
