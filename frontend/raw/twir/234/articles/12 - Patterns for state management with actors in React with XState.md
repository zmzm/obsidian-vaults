# Item 12: Patterns for state management with actors in React with XState

Source: [https://www.typeonce.dev/article/patterns-for-state-management-with-actors-in-react-with-xstate](https://www.typeonce.dev/article/patterns-for-state-management-with-actors-in-react-with-xstate)

Summary:
This article surveys patterns for managing state in React using XState actors, focusing on composability and separation of concerns. It contrasts monolithic actor designs with more granular approaches, such as splitting form and input logic into separate actors, and demonstrates how to wire them together for maintainable, testable state management.

Key takeaways:
- XState actors enable modular, reusable state logic in React.
- Monolithic actors are simple but not scalable for complex forms.
- Splitting logic into parent/child actors improves maintainability.
- Patterns include event forwarding, context sharing, and actor composition.
