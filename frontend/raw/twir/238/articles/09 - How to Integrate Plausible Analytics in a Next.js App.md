Summary:
This guide shows how to integrate Plausible Analytics into a Next.js App Router project in a way that avoids ad blockers by proxying analytics scripts and API requests through your own domain. It also covers tracking page views on client-side navigation and using an enhanced Plausible script for richer analytics.

Key takeaways:
- Proxying analytics scripts via Next.js rewrites reduces the chance of being blocked by ad blockers.
- The guide provides code for a RouteTracker component to track page views on route changes.
- Using a self-hosted, enhanced Plausible script enables advanced analytics features.
- The approach is privacy-friendly and works well with Next.js's routing model.
