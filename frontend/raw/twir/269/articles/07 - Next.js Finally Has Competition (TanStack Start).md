# Item 7: Next.js Finally Has Competition (TanStack Start)

Source: [https://dev.to/elvissautet/nextjs-finally-has-competition-2lg7](https://dev.to/elvissautet/nextjs-finally-has-competition-2lg7)

Summary:
TanStack Start emerges as a compelling alternative to Next.js, emphasizing explicit server/client boundaries, type safety, and developer control. The article contrasts the architectural philosophies and developer experiences of both frameworks, highlighting TanStack Start’s single-file approach and clear separation of concerns versus Next.js’s server/client split and "use client" directive. The analysis is grounded in real-world scenarios and code samples.

Key takeaways:
- Next.js popularized server components but can introduce complexity with server/client boundaries.
- TanStack Start offers explicit server/client code separation, reducing confusion and boilerplate.
- The new framework is still in Release Candidate status but is rapidly maturing.
- The article provides practical, code-level comparisons for common app patterns.

Recommendation: Read fully (for developers choosing a React framework or interested in modern full-stack approaches)
