# Item 5: Accessibility in React: Common Mistakes and How to Fix Them

Source: [https://certificates.dev/blog/accessibility-in-react-common-mistakes-and-how-to-fix-them](https://certificates.dev/blog/accessibility-in-react-common-mistakes-and-how-to-fix-them)

Summary:
This post outlines frequent accessibility mistakes in React apps, such as missing semantics, improper focus management, unlabeled elements, and silent dynamic updates. It emphasizes using correct HTML elements, maintaining logical heading hierarchies, labeling all interactive elements, and leveraging ARIA attributes and useId for proper associations. Practical code examples illustrate both common pitfalls and best practices for making React apps more accessible.

Key takeaways:
- Use semantic HTML elements (button, a, nav, etc.) for accessibility out-of-the-box.
- Maintain logical heading and landmark structure for screen reader navigation.
- Ensure all interactive elements have accessible names and labels.
- Use useId for unique, stable IDs in forms and dynamic content.
