# Item 1: React Router and React Server Components: The Path Forward

Source: [https://remix.run/blog/react-router-and-react-server-components](https://remix.run/blog/react-router-and-react-server-components)

Summary:
This post details the integration of React Server Components (RSC) into React Router, explaining how RSC simplifies and enhances routing, data loading, streaming UI, and code splitting. The new architecture reduces bundler coupling and complexity, making advanced routing features more accessible and flexible. Both traditional loader-based and RSC patterns are supported, offering developers a choice in data and UI management. The migration path is seamless, with no breaking changes for existing users.

Key takeaways:
- React Router’s new RSC-powered Data Mode brings Framework Mode features to lower-level APIs.
- The architecture shift reduces reliance on bundler plugins and custom manifests.
- Both loader-based (Framework Mode) and RSC patterns for data loading and streaming are available.
- Migration to the new system is seamless; existing code will continue to work without changes.
