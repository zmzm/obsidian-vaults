# Item 10: Accessibility essentials every front-end developer should know

Source: [https://martijnhols.nl/blog/accessibility-essentials-every-front-end-developer-should-know](https://martijnhols.nl/blog/accessibility-essentials-every-front-end-developer-should-know)

Summary:
This article outlines essential accessibility practices for React developers, emphasizing semantic HTML, proper form structure, keyboard navigation, focus management, and the use of ARIA attributes. It provides practical advice and code examples for making components accessible, including leveraging native elements and React's useId hook for unique IDs. The article also covers accessible modals and the importance of not disabling focus indicators.

Key takeaways:
- Use semantic HTML and native elements for built-in accessibility.
- Always provide labels for form fields and use unique IDs (e.g., via useId).
- Ensure keyboard navigation and focus indicators are present and visible.
- Prefer native <dialog> for modals, or handle focus management carefully in custom implementations.
