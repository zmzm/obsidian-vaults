# Item 6: RSC for LISP Developers

Source: [https://overreacted.io/rsc-for-lisp-developers/](https://overreacted.io/rsc-for-lisp-developers/)

Summary:
Dan Abramov draws parallels between LISP's code-as-data philosophy and React Server Components (RSC), focusing on the concept of "quoting" code to control where and how it runs. He explains how 'use client' in RSC acts as a boundary, enabling modular composition of server and client logic, though with less metaprogramming flexibility than LISP.

Key takeaways:
- 'use client' in RSC is analogous to LISP's quoting, marking code for client-side execution.
- RSC enables modular composition of server and client code, with the server orchestrating the composition.
- The analogy helps demystify RSC for developers familiar with LISP or functional programming.
