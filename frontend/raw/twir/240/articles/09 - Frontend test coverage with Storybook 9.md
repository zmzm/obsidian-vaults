# Item 9: Frontend test coverage with Storybook 9

Source: [https://storybook.js.org/blog/frontend-test-coverage-with-storybook-9/](https://storybook.js.org/blog/frontend-test-coverage-with-storybook-9/)

Summary:
Storybook 9 introduces integrated frontend coverage reporting, measuring how much of your component code is exercised by stories. It combines interaction, accessibility, and visual tests, highlighting untested states and components. The workflow encourages filling coverage gaps by adding or updating stories, and integrates with common coverage tools for unified reporting.

Key takeaways:
- Storybook 9 tracks coverage across stories, showing which code paths and states are tested.
- Integrates with Vitest, axe-core, and Chromatic for comprehensive UI testing.
- Coverage reports highlight missing stories and untested branches, guiding test improvements.
- AI tools can help generate stories for untested components, accelerating coverage.
