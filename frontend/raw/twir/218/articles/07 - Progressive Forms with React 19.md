# Item 7: Progressive Forms with React 19

Source: [https://rdrn.me/react-forms/](https://rdrn.me/react-forms/)

Summary:
This post explores the new form handling patterns in React 19, focusing on server actions, validation, error handling, and state management. It demonstrates how to use server-side validation with zod, handle errors with useActionState, and maintain form state across submissions. The article provides practical code examples and discusses the tradeoffs of the new approach compared to traditional SPA forms.

Key takeaways:
- React 19 introduces first-class support for server actions and progressive forms.
- useActionState enables seamless error handling and feedback in forms.
- Maintaining form state requires explicit handling, especially after validation errors.
- The new patterns simplify backend-driven forms but introduce new considerations for state and UX.
