# Item 5: AI-Generated UI Is Inaccessible by Default

Source: [https://frontendmasters.com/blog/ai-generated-ui-is-inaccessible-by-default/](https://frontendmasters.com/blog/ai-generated-ui-is-inaccessible-by-default/)

Summary:
The article critiques the accessibility shortcomings of React components generated by AI code tools, noting that most outputs lack semantic HTML and ARIA attributes, resulting in inaccessible UIs. It explains the architectural reasons for these failures, such as training data bias and lack of accessibility tree modeling, and proposes a five-layer enforcement system to ensure semantic correctness. While some tools like Vercel’s v0 are improving, the majority of AI-generated code remains inaccessible without systematic enforcement and verification. The article provides practical examples and highlights the importance of prompt constraints, static analysis, and accessible abstractions.

Key takeaways:
- Most AI-generated React UIs lack semantic structure, making them inaccessible to assistive technologies.
- Failures include missing roles, headings, landmarks, ARIA attributes, and keyboard interactions.
- The root causes are training data bias, visual feedback loops, and lack of accessibility modeling in LLMs.
- Systematic enforcement (prompt constraints, static/runtime checks, accessible abstractions) is necessary for accessible output.
