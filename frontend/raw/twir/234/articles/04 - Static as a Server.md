# Item 4: Static as a Server

Source: [https://overreacted.io/static-as-a-server/](https://overreacted.io/static-as-a-server/)

Summary:
This post explains how React Server Components (RSC) can be used to generate static sites by running server logic at build time, blurring the lines between "server" and "static" frameworks. Modern frameworks like Next.js and Astro support hybrid modes, allowing developers to mix static and server-rendered pages as needed. The author argues that "static" is just a server that runs ahead of time, and hybrid frameworks reduce ecosystem fragmentation and increase flexibility.

Key takeaways:
- RSC can be used for static site generation by running server code at build time.
- Hybrid frameworks allow mixing static and server-rendered routes.
- There’s little value in purely static-only tools compared to hybrid approaches.
- The distinction between "server" and "static" is increasingly irrelevant in modern React workflows.
