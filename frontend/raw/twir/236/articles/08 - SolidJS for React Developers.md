# Item 8: SolidJS for React Developers

Source: [https://marmelab.com/blog/2025/05/28/solidjs-for-react-developper.html](https://marmelab.com/blog/2025/05/28/solidjs-for-react-developper.html)

Summary:
This article introduces SolidJS to React developers, highlighting similarities in syntax but emphasizing key architectural differences, such as the absence of a virtual DOM and the use of fine-grained reactivity via signals. It explains how SolidJS handles control flow, state, and effects differently, and provides insights into its internal observer pattern.

Key takeaways:
- SolidJS uses signals and fine-grained reactivity instead of a virtual DOM.
- Control flow is handled via components like <Show>, <Switch>, and <For> rather than JavaScript conditionals in JSX.
- Effects automatically track dependencies; state is accessed via getter functions.
- The observer pattern underpins Solid’s reactivity model.
