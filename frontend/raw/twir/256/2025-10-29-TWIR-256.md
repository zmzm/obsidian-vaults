# This Week in React #256

## TL;DR

- Tanner Linsley critiques framework directives like "use client" and "use server" — they blur the platform/language boundary and create tooling friction without a shared spec
- TanStack Start Middleware allows pre/post-processing of server functions on both client and server, enabling observability and single-flight mutations
- Why developers are leaving Next.js for TanStack Start: complexity, Vercel lock-in, versus explicitness and type-safe routing
- React vs Remix diverging: React embraces complexity for stability and capability; Remix 3 breaks away toward simplicity and Web Platform alignment
