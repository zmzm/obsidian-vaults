# Item 10: The Limits of RSC: A Practitioner's Journey

Source: [https://www.nirtamir.com/articles/the-limits-of-rsc-a-practitioners-journey](https://www.nirtamir.com/articles/the-limits-of-rsc-a-practitioners-journey)

Summary:
This post shares a real-world account of adopting React Server Components (RSC) and encountering limitations, particularly with patterns like infinite scrolling that require persistent client-side state. The author describes how RSC’s statelessness and server-only data fetching make certain interactive features cumbersome or inefficient, leading to practical compromises and hybrid solutions.

Key takeaways:
- RSC is well-suited for static or mostly server-rendered content.
- Features needing persistent client state (e.g., infinite scroll) are challenging with RSC.
- Workarounds often involve hybrid approaches or client-side state management.
- RSC’s promise is real, but not a universal fit for all app patterns.
