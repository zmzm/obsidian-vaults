# Item 13: Why use React?

Source: [https://adactio.com/journal/22265](https://adactio.com/journal/22265)

Summary:
Jeremy Keith reflects on the reasons developers choose React, often citing inertia, developer experience, or organizational mandates rather than user experience. He argues that while React’s authoring model is compelling, shipping React to the browser imposes a cost on users. The article encourages using React for server-side rendering or authoring, but minimizing or eliminating client-side React when possible, and considering alternatives like Astro or Preact for better frontend performance.

Key takeaways:
- React adoption is often driven by developer comfort or organizational standardization.
- Shipping React to browsers increases client-side overhead; SSR or alternatives can mitigate this.
- Encourages evaluating whether client-side React is necessary for each project.
- Highlights the growing power of native browser capabilities and the importance of user-centric choices.
