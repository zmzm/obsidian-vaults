# Item 13: Front-end testing at Preply: shifting left towards component testing

Source: [https://www.columkelly.com/blog/use-optimistic](https://www.columkelly.com/blog/use-optimistic)

Summary:
Preply describes its transition from heavy reliance on end-to-end (E2E) tests to a more balanced approach emphasizing component testing, especially for React and React Native. The company established an Engineering Excellence Group, adopted tools like Vitest and Storybook, and implemented accessibility and visual regression testing. The shift aims to improve long-term quality, developer experience, and maintainability across web and mobile platforms.

Key takeaways:
- Shift-left strategy: moving from E2E to component and integration testing for better coverage and maintainability.
- Adoption of modern tools (Vitest, Storybook, Chromatic) and accessibility checks in CI.
- Emphasis on documentation, onboarding, and shared quality standards across teams.
- Custom solutions for React Native component testing are in development.
