# Item 5: React State Management in 2025: What You Actually Need

Source: [https://www.developerway.com/posts/react-state-management-2025](https://www.developerway.com/posts/react-state-management-2025)

Summary:
This opinionated article argues that most React apps do not need a dedicated state management library, and that many state concerns are now better handled by specialized solutions or React’s built-in features. It emphasizes understanding the real reasons for state management pain—often related to remote data fetching and caching—and recommends TanStack Query for data management. The author cautions against switching libraries without diagnosing the actual problem.

Key takeaways:
- Most state management pain comes from remote data handling, not local state.
- TanStack Query (React Query) is recommended for remote data needs, replacing much of the boilerplate from Redux.
- Avoid switching state libraries without clear, measured reasons; focus on understanding your app’s true state needs.
- Built-in React features and targeted libraries often suffice for modern apps.
