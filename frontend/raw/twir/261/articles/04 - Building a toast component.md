# Item 4: Building a toast component

Source: [https://emilkowal.ski/ui/building-a-toast-component](https://emilkowal.ski/ui/building-a-toast-component)

Summary:
Emil Kowalski details the design and engineering behind Sonner, a popular React toast library known for its smooth animations and developer experience. The post covers naming, animation techniques (CSS transitions vs. keyframes), stacking logic, swipe-to-dismiss gestures, and expand-on-hover features, with code snippets and rationale for each decision. The focus is on balancing UX polish with ease of integration and customization.

Key takeaways:
- Sonner’s unique animations and gestures contributed to its rapid adoption.
- Uses CSS transitions for interruptible, smooth stacking animations.
- Implements swipe and expand behaviors for better usability.
- Emphasizes developer experience and maintainable API design.

Recommendation: Read fully (valuable for those building or evaluating notification/toast components)
