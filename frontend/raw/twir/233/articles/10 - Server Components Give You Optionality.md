# Item 11: Server Components Give You Optionality

Source: [https://saewitz.com/server-components-give-you-optionality](https://saewitz.com/server-components-give-you-optionality)

Summary:
This essay argues that React Server Components (RSC) are additive, not prescriptive, giving developers flexibility to choose where and when code runs (server, client, build-time). It contrasts CSR, SSR, and RSC, highlighting how RSC enables static, server-only, and client-only code, reducing client bundle size and improving UX. The article positions RSC as a powerful tool for advanced teams seeking fine-grained control over rendering and performance.

Key takeaways:
- RSCs let you choose execution environment per component: server, client, or build-time.
- Server-only components reduce client bundle size and improve page durability.
- The model supports static export, SSR, and client interactivity, similar to Astro’s islands but with React’s ecosystem.
- RSCs enable “tree-shaking by design,” sending only necessary code to the client.
