# Item 6: Migrating from Radix UI to Base UI: Step-by-Step Guide

Source: [https://shadcnstudio.com/blog/migrate-from-radix-ui-to-base-ui](https://shadcnstudio.com/blog/migrate-from-radix-ui-to-base-ui)

Summary:
This guide details the migration process from Radix UI to Base UI for shadcn/ui users, explaining the rationale behind the shift and providing a step-by-step migration plan. Base UI, developed by the teams behind Radix, Floating UI, and Material UI, offers a unified package, modern API (moving from asChild to render prop), and improved dependency management. The guide covers migration steps, common pitfalls, and the benefits of adopting Base UI for long-term maintainability.

Key takeaways:
- Base UI consolidates primitives into a single package with a modern, explicit API.
- Migration involves updating component usage, handling API differences, and cleaning up dependencies.
- The new render prop pattern is clearer and more predictable than asChild.
- Base UI is backed by experienced teams and aims for long-term stability.
