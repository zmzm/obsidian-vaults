# Item 8: Search Params Are State

Source: [https://tanstack.com/blog/search-params-are-state](https://tanstack.com/blog/search-params-are-state)

Summary:
Tanner Linsley argues that URL search parameters should be treated as first-class, type-safe state, not just strings. He critiques current ad-hoc approaches and shows how TanStack Router embeds search param schemas directly in route definitions, enabling validation, inference, and safe updates, thus preventing fragmentation and bugs.

Key takeaways:
- Search params are often handled with brittle, manual parsing and lack type safety.
- TanStack Router ties search param schemas to routes, enabling type-safe, validated, and coordinated usage.
- This approach prevents schema drift and makes navigation and deep linking more robust.
