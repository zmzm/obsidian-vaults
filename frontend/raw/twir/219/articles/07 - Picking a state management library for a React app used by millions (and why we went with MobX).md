# Item 15: Picking a state management library for a React app used by millions (and why we went with MobX)

Source: [https://www.photoroom.com/inside-photoroom/picking-a-state-management-library-why-we-went-with-mobx-](https://www.photoroom.com/inside-photoroom/picking-a-state-management-library-why-we-went-with-mobx-)

Summary:
Photoroom migrated its large-scale React app from useContext and Constate to MobX for state management, after evaluating Zustand and Redux. MobX’s automatic derivation and fine-grained reactivity reduced unnecessary re-renders and simplified state logic, especially for real-time editing. The transition improved testing and developer experience, though hiring experienced MobX developers proved challenging.

Key takeaways:
- MobX enables automatic state derivation and granular reactivity.
- Migration from useContext reduced boilerplate and improved performance.
- Testing became easier due to class-based state.
- Downsides include a smaller talent pool for MobX expertise.
