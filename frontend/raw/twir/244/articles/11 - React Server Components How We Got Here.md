# Item 11: React Server Components: How We Got Here

Source: [https://www.epicreact.dev/react-server-components-how-we-got-here-zcuxn](https://www.epicreact.dev/react-server-components-how-we-got-here-zcuxn)

Summary:
Kent C. Dodds traces the evolution from MPAs to SPAs, then to progressive enhancement frameworks (like Remix), and finally to React Server Components (RSC). Each step addressed previous shortcomings, with RSCs restoring component-based composition and eliminating custom network management code. The article highlights how RSCs combine the best aspects of prior paradigms—simplicity, interactivity, progressive enhancement, and composability—while introducing a new mental model for developers.

Key takeaways:
- RSCs move the composition boundary back to components and remove network management boilerplate.
- Each web paradigm (MPA, SPA, PESPAs, RSC) solved previous problems but introduced new ones.
- RSCs offer optimal performance, progressive enhancement, and clear server/client separation.
- Adopting RSCs requires learning a new mental model, but the benefits are substantial.
