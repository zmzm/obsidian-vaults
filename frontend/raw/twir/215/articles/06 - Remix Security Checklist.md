# Item 6: Remix Security Checklist

Source: [https://blog.arcjet.com/remix-security-checklist/](https://blog.arcjet.com/remix-security-checklist/)

Summary:
This checklist covers best practices for securing Remix (React-based) applications, focusing on dependency management, module boundaries, environment variables, and authentication. It emphasizes defense-in-depth, proper placement of sensitive logic, and careful handling of environment variables and cookies. The article also highlights Arcjet’s security SDK for Remix.

Key takeaways:
- Keep dependencies updated and review third-party packages for supply chain risks.
- Avoid module-level side effects; wrap sensitive logic in loaders/actions to ensure server-only execution.
- Never expose secrets or sensitive environment variables to the client.
- Use Remix’s cookie/session utilities to securely manage authentication and authorization.
