# Item 5: All kinds of state management in React

Source: [https://www.typeonce.dev/article/all-kinds-of-state-management-in-react](https://www.typeonce.dev/article/all-kinds-of-state-management-in-react)

Summary:
This article categorizes different types of state in React, emphasizing that not all state requires a library—especially with modern React and frameworks like Next.js. It demonstrates how URL state can be managed via search parameters and browser history, reducing the need for local state. The article provides practical code examples for managing state through the URL and integrating with server and client components.

Key takeaways:
- React 19 and frameworks like Next.js provide built-in solutions for many state management needs.
- URL state is ideal for simple, shareable values and browser navigation.
- Hooks and new APIs (like use and useSearchParams) make it easier to synchronize state with the URL.
- Evaluate if a state management library is necessary before adding complexity.
