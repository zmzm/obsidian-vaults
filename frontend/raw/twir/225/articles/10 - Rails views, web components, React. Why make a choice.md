# Item 11: Rails views, web components, React. Why make a choice?

Source: [https://www.aha.io/engineering/articles/rails-views-web-components-react](https://www.aha.io/engineering/articles/rails-views-web-components-react)

Summary:
This article discusses how Aha! combines Rails views, ViewComponents, web components, and React to leverage the strengths of each technology. It argues against the “one true way” mindset and illustrates how a hybrid approach can yield more maintainable and flexible applications. Real-world scenarios from Aha!’s codebase are used to show when each technology is most appropriate.

Key takeaways:
- Rails views are used for simple, fast-to-develop features.
- ViewComponents provide reusable, encapsulated UI patterns in Ruby.
- Web components bridge Rails and React, enabling shared functionality.
- React is reserved for highly interactive, complex UI features.
