# Item 8: Concurrent Optimistic Updates in React Query

Source: [https://tkdodo.eu/blog/concurrent-optimistic-updates-in-react-query](https://tkdodo.eu/blog/concurrent-optimistic-updates-in-react-query)

Summary:
This article discusses the challenges of implementing optimistic UI updates in React Query, especially when multiple updates occur concurrently. It highlights the need to replicate server logic on the client for accurate optimistic updates and addresses issues like query cancellation to prevent inconsistent UI states. The post provides practical code examples and explains edge cases where optimistic updates can fail or lead to race conditions.

Key takeaways:
- Optimistic updates require duplicating server logic on the client, which can be complex and error-prone.
- Query cancellation is essential to avoid stale data overwriting optimistic updates.
- Concurrent mutations can introduce race conditions and inconsistent UI if not managed carefully.
