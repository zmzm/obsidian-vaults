# Item 12: Make FormData and input names type-safe in React

Source: [https://www.typeonce.dev/article/make-form-data-and-input-names-type-safe-in-react](https://www.typeonce.dev/article/make-form-data-and-input-names-type-safe-in-react)

Summary:
This article addresses the lack of type safety between form input names and FormData access in React, especially when using useActionState. It proposes a pattern using TypeScript generics and custom components to enforce type-safe input names and FormData access, reducing the risk of runtime errors due to typos or mismatches.

Key takeaways:
- Default FormData access is not type-safe; typos can cause silent bugs.
- Custom components and generics can enforce input name consistency at compile time.
- The pattern scales to larger forms and improves maintainability.
- Applies to both input elements and FormData extraction.
