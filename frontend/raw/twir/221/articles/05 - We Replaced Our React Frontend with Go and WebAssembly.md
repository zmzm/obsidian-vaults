# Item 7: We Replaced Our React Frontend with Go and WebAssembly

Source: [https://dagger.io/blog/replaced-react-with-go](https://dagger.io/blog/replaced-react-with-go)

Summary:
Dagger Cloud migrated its frontend from React to a Go/WebAssembly (WASM) stack to unify codebases and improve performance. The switch enabled code sharing with their terminal UI and addressed performance bottlenecks in large-scale data visualization. The team overcame challenges like WASM memory limits and lack of mature Go UI libraries, ultimately achieving a more maintainable and performant solution tailored to their needs.

Key takeaways:
- Go/WASM enabled code reuse and improved performance for complex UIs.
- Memory management and ecosystem maturity were primary challenges.
- Go-app provided a React-like component model, easing the transition.
- The approach is niche but may inspire teams with similar constraints.
