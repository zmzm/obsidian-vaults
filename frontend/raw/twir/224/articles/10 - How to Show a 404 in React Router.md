Summary:
This tutorial covers multiple approaches to handling 404 (Not Found) errors in React Router, including using error boundaries and catch-all (“splat”) routes. It explains how to trigger 404s for both unmatched routes and missing resources, and how to customize error messages and status codes.

Key takeaways:
- Error boundaries can catch unmatched routes and resource-level 404s.
- Splat routes (/*) provide a flexible way to handle generic 404s.
- Loaders can set status codes and fetch data for error pages.
- Custom error boundaries allow for tailored error messages per route.
