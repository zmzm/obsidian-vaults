# Framework Conventions in the AI Era

This case study captures a practical argument for frameworks as convention systems in hybrid human-and-agent development.

## Context

- The experiment removed Next.js in favor of plain HTML and JavaScript while using an AI agent heavily in the implementation loop.
- The goal was to test whether agent assistance reduces the need for framework structure.

## What Broke Down

- Code organization lost consistency once framework conventions disappeared.
- Human expectations and agent output drifted apart more easily.
- The cost showed up in maintainability and coordination, not just in missing features.

## Why It Matters

- This is not just a Next.js story; it is a broader engineering-practice story about explicit conventions.
- It is especially relevant to this vault because the same logic applies to LLM-maintained systems and collaborative codebases.

## Main Lesson

- In AI-assisted projects, frameworks often earn their keep by enforcing shared structure rather than by hiding low-level APIs.
