# Item 11: React Server Components in React Router – First Impressions

Source: [https://raphaelbronsveld.com/blog/rsc-in-rr-first-impressions](https://raphaelbronsveld.com/blog/rsc-in-rr-first-impressions)

Summary:
The author experiments with React Server Components (RSC) in React Router, describing the new setup requirements (multiple entry points, Vite plugin) and differences from traditional loaders. Early impressions are positive, with smooth migration for static routes, but some uncertainty remains about layout patterns and deployment. The article also highlights performance considerations and the need to use Suspense boundaries thoughtfully.

Key takeaways:
- RSC support in React Router is experimental but functional for static content.
- New build setup requires multiple entry points and a Vite plugin.
- Loader functions may shift toward managing status codes, with RSCs handling data fetching directly.
- Performance and Suspense boundary placement require attention.
