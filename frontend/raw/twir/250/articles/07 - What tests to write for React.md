# Item 7: What tests to write for React

Source: [https://www.franciscomoretti.com/blog/what-tests-to-write-for-react](https://www.franciscomoretti.com/blog/what-tests-to-write-for-react)

Summary:
This article reviews modern testing strategies for React, advocating for the "Testing Trophy" model over the traditional Test Pyramid. The Testing Trophy emphasizes integration tests as the most valuable, supported by static analysis, with fewer unit and end-to-end tests. The post provides practical examples for each test type and recommends tools like TypeScript, ESLint, Vitest, and Playwright.

Key takeaways:
- Integration tests should be the primary focus for React apps, providing the best confidence-to-effort ratio.
- Static analysis (type checking, linting) forms a strong foundation for catching issues early.
- Unit tests are useful for isolated logic; end-to-end tests are valuable but costly to maintain.
- Modern tools make integration and static testing more practical and efficient.
