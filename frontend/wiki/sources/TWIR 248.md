# TWIR 248

TWIR #248 connects compiler enforcement with compatibility diagnostics and framework-independent RSC support.

## Summary

- The issue notes that the React Compiler rule becomes enabled by default in `eslint-plugin-react-hooks`.
- It also captures explicit incompatibility errors for popular libraries.
- On the RSC side, it includes Forket as a framework-light way to use Server Components.

## Why This Source Matters

- It materially supports both the compiler branch and the vault's interest in explicit lint-driven constraints.
- It also broadens the evidence that RSC support was moving outside the largest frameworks.

## Caveats

- The issue is more about ecosystem direction than settled best practice.
