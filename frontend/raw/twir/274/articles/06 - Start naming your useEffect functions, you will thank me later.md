# Item 6: Start naming your useEffect functions, you will thank me later

Source: [https://neciudan.dev/name-your-effects](https://neciudan.dev/name-your-effects)

Summary:
Naming your useEffect function expressions clarifies intent and improves code readability, especially in components with multiple effects. This simple change makes it easier to understand and review code, reducing cognitive load during maintenance and code review.

Key takeaways:
- Anonymous useEffect callbacks obscure the purpose of each effect.
- Named function expressions in useEffect make the code self-documenting.
- Improves onboarding, review, and debugging, especially in large components.
- The change is purely syntactic and has no runtime impact.
