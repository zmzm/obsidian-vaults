# Item 4: The time is right for a DOM templating API

Source: [https://justinfagnani.com/2025/06/26/the-time-is-right-for-a-dom-templating-api/](https://justinfagnani.com/2025/06/26/the-time-is-right-for-a-dom-templating-api/)

Summary:
This post advocates for adding a declarative templating API to the web platform, arguing that templating is foundational to modern web development but currently only available via frameworks. The author outlines the benefits of native templating—such as ergonomics, security, performance, and static analysis—and discusses why now is an opportune moment for standardization. The piece also surveys past attempts and the convergence of userland solutions as a basis for new platform APIs.

Key takeaways:
- Declarative templating is ubiquitous and brings major DX, UX, and security benefits, but is missing from the native DOM API.
- The lack of a standard solution increases bundle sizes, onboarding friction, and framework complexity.
- Modern frameworks have converged on similar patterns, making standardization more feasible.
- There is growing demand from both framework and "vanilla" developers for ergonomic, secure DOM manipulation.
