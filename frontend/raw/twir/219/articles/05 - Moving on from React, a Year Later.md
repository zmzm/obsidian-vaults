# Item 11: Moving on from React, a Year Later

Source: [https://kellysutton.com/2025/01/18/moving-on-from-react-a-year-later.html](https://kellysutton.com/2025/01/18/moving-on-from-react-a-year-later.html)

Summary:
A year after migrating from React to a server-rendered Rails/Stimulus stack, the author reflects on reduced complexity, improved testability, and faster performance. The team found that server-rendered pages with minimal JavaScript led to more maintainable code and fewer flaky tests, with no significant loss in user experience. The article argues that JavaScript-heavy SPAs introduce unnecessary overhead for many business applications.

Key takeaways:
- Server-rendered apps can be faster, simpler, and easier to test than SPAs.
- Minimal JavaScript reduces maintenance and regression risk.
- The team experienced better developer productivity and reliability post-migration.
- Not all apps benefit from the complexity of a React SPA.
