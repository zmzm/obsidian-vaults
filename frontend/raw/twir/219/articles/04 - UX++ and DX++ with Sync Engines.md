# Item 10: UX++ and DX++ with Sync Engines

Source: [https://www.carlassmann.com/blog/improve-ux-dx-with-sync-engines](https://www.carlassmann.com/blog/improve-ux-dx-with-sync-engines)

Summary:
The article discusses the complexity of managing remote state in modern React apps, especially with tools like TanStack Query. It highlights issues with loading states, cache invalidation, and optimistic UI, arguing that current solutions lead to complex, “waiting” user experiences. The author advocates for sync engines—local-first architectures that keep a client-side copy of data and sync with the server asynchronously—to improve both user and developer experience.

Key takeaways:
- Remote state management introduces complexity and suboptimal UX.
- Manual cache invalidation and optimistic UI are error-prone.
- Local-first sync engines can eliminate network roundtrips from critical paths.
- Rethinking data flows can simplify code and improve responsiveness.
