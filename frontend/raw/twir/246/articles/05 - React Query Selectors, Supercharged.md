# Item 5: React Query Selectors, Supercharged

Source: [https://tkdodo.eu/blog/react-query-selectors-supercharged](https://tkdodo.eu/blog/react-query-selectors-supercharged)

Summary:
This article delves into the `select` option in React Query, which allows components to subscribe only to specific slices of query data, minimizing unnecessary re-renders. It explains how `select` provides fine-grained subscriptions, similar to selectors in Redux, and discusses type inference and best practices for abstraction in TypeScript. The article also contrasts React Query's approach with other state managers like Zustand.

Key takeaways:
- The `select` option in React Query enables components to re-render only when selected data changes.
- Fine-grained subscriptions improve performance, especially with large or frequently changing data.
- Type inference works seamlessly with `select`, but abstraction requires careful TypeScript typing.
- Structural sharing in React Query ensures efficient updates without referential instability.
